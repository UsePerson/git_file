Data System Structure
/bin 放 single-user mode 可用的基本指令, ex: cat, ls, cp
/sbin Essential system binaries
/boot Boot loader files
/dev 放置 device 相關的檔案，ex 滑鼠、鍵盤
/etc 放置 system 相關的檔案
/home user 的家目錄
/root sudo 的家目錄
/lib Libraries essential for the binaries in /bin/ and /sbin/
/lib64 system 函式庫和 kernel 函式庫 (主要放64位元)
/proc map 到 process table, 人們覺得它好棒棒, 所以加了很多的東西 ex cpuinfo, device infomation..., 所以出現了一些問題
/sys 因為 /proc 那邊之前加了太多東西， 所以為了解決那個問題，所以多了 /sys 來 hold system information
/usr 是 Secondary hierarchy for read-only user data，包含大多數 user utilities and applications
/media 可移動式 device 的臨時安裝目錄
/mnt 臨時掛載目錄
/opt 包含附加應用程序。
/srv 此系統服務的特定於站點的數據，例如Web服務器的數據和腳本，FTP服務器提供的數據以及版本控制系統的存儲庫。
/tmp 暫存的資料夾，包含由 system 和 user 創建出來的檔案，這些檔案通常在重新開機就被刪掉ㄌ
    掛載 : 也就是將軟體上載到虛擬光碟機，並安裝的意思。
